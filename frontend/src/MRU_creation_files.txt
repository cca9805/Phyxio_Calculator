Archivos implicados para crear el subtema MRUV (a partir de MRU)

1) pages/cinematica/MRUV.jsx
   - Acción: crear un nuevo archivo copiando pages/cinematica/MRU.jsx y renombrando 
   meta.id a "mruv", meta.title a "MRUV" y meta.intro/tldr según corresponda.
   Este es el componente de la página del subtema.

2) components/SubtopicLayout.jsx
   - Acción: no es necesario modificar la estructura principal, 
   pero asegúrate de que SubtopicLayout pueda obtener datos para 
   id="mruv" (fetch localStorage o /api). 
   Si usas un índice local, añade la entrada en los archivos de datos abajo.

3) src/App.jsx (o el archivo donde está el <Routes>)
   - Acción: importar el nuevo componente MRUV y añadir una 
   <Route path="/cinematica/mruv" element={<MRUV />} /> 
   o usar una ruta dinámica /cinematica/:subtopicId si ya existe.

4) pages/Cinematica.jsx
   - Acción: añadir una tarjeta/enlace a MRUV en la lista de subtemas de 
   Cinemática para que los usuarios puedan navegar desde la sección.

5) data/cinematicaNivel2.js
   - Acción: (si existe un índice por sección) añadir metadata para mruv: 
   { id: 'mruv', title: 'MRUV', intro: '...' , palette: '...' , order: N, 
   sections: [...] }
   - Detalle: sections puede contener el array de secciones que 
   SubtopicLayout mostrará en el Accordion.

6) data/subtemasFisica.js o data/sidebarItems.js
   - Acción: añadir la entrada del nuevo subtema en el listado global/índice 
   para que aparezca en menús o navegación lateral.

7) components/NavbarSidebar.jsx (o el archivo de navegación relevante)
   - Acción: si el sidebar no lee el índice de data, añadir manualmente el <a> o 
   el item que apunte a /cinematica/mruv.

8) components/Accordion.jsx y components/MarkdownRenderer.jsx
   - Acción opcional: verificar que soportan los tipos de sección 
   (markdown, formulas, simulador) que vayas a incluir en la metadata de MRUV.

9) styles/nivel3.css (opcional)
   - Acción: añadir o ajustar estilos si el nuevo subtema necesita reglas específicas.

Flujo recomendado (pasos concretos):
- Crear pages/cinematica/MRUV.jsx copiando MRU.jsx y actualizar meta.
- Añadir entrada en data/cinematicaNivel2.js y/o data/subtemasFisica.js (id 'mruv').
- Registrar ruta en src/App.jsx o confirmar que la ruta dinámica existente 
    (/cinematica/:subtopicId) funciona.
- Añadir enlace/target en pages/Cinematica.jsx o en el sidebar para que se muestre.
- Abrir la nueva ruta en el navegador y verificar que SubtopicLayout carga las 
    secciones desde metadata.

Notas:
- Si SubtopicLayout hace fetch a /api/pages/:id y no hay backend, 
    guarda el JSON en localStorage bajo 'phyxio_pages' con la entrada para 'mruv' 
    (o adapta SubtopicLayout para usar los archivos data/ locales).
- Mantén IDs en minúsculas y sin espacios (slug) para consistencia.
